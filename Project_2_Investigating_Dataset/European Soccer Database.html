Data Wrangling¶

  1. In this section I will use SQL Syntax (Using sqlalchemy package) to retrieve all tables from the SQL DataBase file then concatenate the tables using pandas, since pandas provide more fine grain options in merging tables.
  2. Choose Columns for analysis and inspect missing values.
  3. Create target column for the match outcome to state weather the home-team win, away-team win or draw.
  4. Impute missing values for the player dataframe using KNNImputer provided by scikit-learn package for advanced imputation techniques.

#Creating a target Column

#Create a mask for the winning stats
home_win = df_matches['home_team_goal'] > df_matches['away_team_goal']
away_win = df_matches['home_team_goal'] < df_matches['away_team_goal']
draw = df_matches['home_team_goal'] == df_matches['away_team_goal']

#Create a list of conditions for the winning team along with the mapping values
conditons_target = [home_win, away_win, draw]
choice_target = ['home_win', 'away_win', 'draw']

#Create a target column specifiying the winning stats( home_win , away_win, draw)
df_matches['target'] = np.select(conditons_target, choice_target, default=np.nan)
df_matches.drop(['home_team_goal', 'away_team_goal'], axis=1,inplace=True)
df_matches.columns = ['country', 'league', 'season', 'team', 'target']

Impute Missing Values Using KNN-Imputer for the player DataFrame¶

In [11]:
#Take 1672 samples from complete df_player data (same number of missing values*2)
not_null_sample = df_player.loc[~df_player.potential.isnull()].iloc[:836*2]
is_null = df_player.loc[df_player.potential.isnull()]

#Concatenate the data the contains null values and the does not contain null values vertically
#We don't impute missing values on the complete data 
#Imputation algorithm is computationally intensive compared to filling missing values with mean or median
data_to_impute = is_null.append(not_null_sample)
data_to_impute_index = pd.DataFrame(data_to_impute.index.to_list(), columns=['index'])
In [12]:
#Instantiate KNNImpute from sklearn library
imputer = KNNImputer()

#fit and transform the imputer on the data
df_player_imputed = imputer.fit_transform(data_to_impute)
df_player_imputed = pd.DataFrame(df_player_imputed, columns=df_player.columns)
df_player_imputed = pd.concat([df_player_imputed, data_to_impute_index], axis=1).set_index('index')

#Update the original data with the imputed values
df_player.update(df_player_imputed)

We can infer that overall_rating & potential columns have almost similar distribution so I will create one feature by creating the average element wise. Height and weight can be combined in one feature by calculating the body mass index (BMI)

$${BMI} = Weight / Height^{2}$$

Where Weight & Height are in metric units

In [19]:
#Create a new data frame for players with only 2 features described above

Conclusions¶

First Question:¶

As for the first Question we can infer that all countries & leagues have a higher home-win than away-win Probability, while not all teams have home-win higher than away-win. As a conclusion we can say that biting against home country or league might be a bad idea based on probabilities, while it might be good idea to bit against home country depending on the playing team.

Body Mass Index have a slight correlation with the rating of the player, so it provides no effect on the player rating, As a conclusion we need to extract other features from the database that can explain the rating of the player.
